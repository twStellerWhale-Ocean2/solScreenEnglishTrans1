# I. 主旨目的

定義「桌面即時查詢工具」這一**系統類型**的技術選型 Profile：常駐使用者桌面背景、以全域熱鍵喚起、完成單次查詢後即退回背景的輕量工具（查詢對象可為畫面內容、剪貼簿、選取文字等）。於 design.md ＜I 方案設計.C 組態設定.(A) 技術選型＞宣告；一經宣告即綁定本契約三件事：(1) 該類型**最低能力清單**、(2) **不干擾介面水準 bar**、(3) **點名強制 techItem**（依查詢輸出型態）。

本契約是「該類系統該長什麼樣」的 normative 單一事實來源；GATE／design 引用本契約、不另抄一份。

# II. 參考準備

* **適用場景**：使用者於全螢幕情境（遊戲、影片、簡報、閱讀）中，需要不中斷主要活動、快速查得輔助資訊的桌面工具——如畫面翻譯、螢幕取詞、快速計算、色彩擷取。
* **公開參照（不重造輪子）**：Windows 系統匣常駐應用慣例（tray app pattern）、全域熱鍵慣例（`RegisterHotKey`）、螢幕區域選取慣例（Snipping Tool／PowerToys 之變暗遮罩＋橡皮筋選框）、Windows 11 Fluent Design（type ramp／圓角／acrylic 層次）、WCAG 對比可近用性。
* **三層串成鏈**：techApp（本層＝系統類型）→ 點名強制 techItem（元件，如語音合成）→ 由 techStack（桌面平台）承載。

# III. 內容程序

## A. 最低能力清單（逐項對照）

* **常駐輕量**：以系統匣常駐背景，閒置時記憶體占用 <100MB、CPU 近 0；不開常駐主視窗。
* **喚起即時**：全域熱鍵按下至介面（遮罩／視窗）出現 <300ms；查詢流程全程不需滑鼠離開主活動視窗以外的操作。
* **零輸入干擾**：熱鍵註冊採系統標準 API（`RegisterHotKey`），**禁用低階鍵盤 hook**（`SetWindowsHookEx WH_KEYBOARD_LL`）除非設計明確要求並經 USR 核准——不得拖慢全系統輸入。
* **隨時可逃**：任一階段 `ESC` 立即取消／關閉，回到喚起前狀態；不留殘影視窗。
* **明確錯誤降級**：外部依賴（API、網路、金鑰）失效時給出明確可讀錯誤與下一步指引，不當機、不無聲失敗。
* **金鑰安全**：外部服務金鑰一律取自環境變數或 OS 憑證保管，程式檔、設定檔與 repo 不落地金鑰。
* **多螢幕與 DPI 正確**：遮罩、選區、結果視窗在多螢幕與非 100% DPI 縮放下座標對位正確。

## B. 不干擾介面水準 bar（逐頁判讀；反「小工具粗糙感」）

* **喚起介面極簡**：遮罩／選框只承載「選取」一件事——變暗底＋高對比選框＋一行操作提示；禁塞工具列、選項、廣告位。
* **結果視窗即讀即走**：單卡片呈現、資訊分區清楚（主結果／輔助資訊／動作鈕）、預設出現於選區旁不遮擋原文；`ESC` 或點擊外部即關。
* **視覺融入 OS**：遵循宿主 OS 設計語言（Windows＝Fluent：Segoe UI 字階、4px 間距格、圓角、暗色支援）；禁自創突兀配色與非標準控制項。
* **狀態可見**：查詢進行中有明確進度指示（骨架屏或 spinner＋文字）；完成、失敗狀態一目了然。
* **明列不合格樣式（出現即回修）**：喚起延遲可感（>500ms）、選區與實際截圖錯位、結果視窗遮住原選區、無 loading 狀態的長白等、錯誤只寫 "Error"、tray 圖示無選單。

## C. 點名強制 techItem（依輸出型態）

* 查詢輸出含「發音／朗讀」者：必含 [techItem語音合成]。
* 其餘 techItem 依 design 各層落地（引各該 techItem 契約）。

## D. 驗收綁定

* **GATE §5 鏡頭A**：逐項對照本契約 §A 最低能力清單。
* **GATE §5 鏡頭C**：逐頁以本契約 §B 不干擾介面 bar 判讀，不合格項回修。
* **視覺基座**：桌面原生應用不錨 [hmiIntf通用視覺規範]（MD3 web 基座），改錨宿主 OS 設計語言（本契約 §B）；可近用性維持 WCAG 對比精神。
